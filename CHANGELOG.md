# Changelog

## v1.5.1

- 修复 v1.5.0 中 `daily_thoughts` 表没有被创建的问题。
- `/api/state` 查询前会先调用 `init_db()`，避免旧数据库缺表导致 500。
- `/api/thoughts` 写入前也会确保数据库结构已初始化。

## v1.5.0

- 新增 `daily_thoughts` 表，用于记录每天多条小感想。
- 下板新增「今天的小感想」输入区和列表。
- 上板新增「昨天的小感想」列表。
- 小感想支持删除。
- 复制今日存档时会包含今天/昨天的小感想。

## v1.4.4

- DDL 进度条从每 30 秒更新一次改为每 1 秒更新一次。
- 进度条动画改为 1 秒线性过渡，看起来更平滑。
- README 新增“调整 DDL 进度条刷新频率”说明。

## v1.4.3

- 保存“写给明天的猫猫留言”成功后，不再使用浏览器原生 `alert`。
- 改为页面底部 toast：「已保存给明天喵」。

## v1.4.2

- 修复自定义弹窗里的日期 / 时间原生图标在暗色模式下仍然发黑的问题。
- 弹窗 date / time 输入框改成自定义 📅 / ⏰ 按钮。
- 补充 `--red-text` CSS 变量，减少 IDE 提示。

## v1.4.1

- 修复 v1.4.0 自定义弹窗初始化太早导致前端 JS 中断的问题。
- 数据库无需改动，只需覆盖 `static/index.html`。

## v1.4.0

- 新增猫猫自定义弹窗 modal。
- 替换修改 DDL、重新启动任务、继续处理过期任务中的原生 `prompt()`。
- 修改 DDL / 重新设置 DDL 时使用网页内 `date` / `time` 输入框，不再需要手动输入。
- 替换部分删除确认的原生 `confirm()`。
- 保留 toast 轻提示。

## v1.3.8

- 未完成 DDL 增加「修改 DDL」按钮。
- 可以修改截止日期和截止时间。
- 日期留空时可以清除 DDL，让任务保留在待办里但不再倒计时。
- 修改成功后显示 toast：「DDL 已修改喵」。
- 新增后端接口：`/api/tasks/<task_id>/due`。

## v1.3.7

- 新增 toast 轻提示。
- 添加 DDL 成功后显示：「DDL 已添加喵」。
- 记录完成成功后显示：「已记录完成喵」。
- 避免手机端提交后没有反馈、误以为卡住。

## v1.3.6

- 已完成任务重新启动时，如果原 DDL 还没过，会保留原截止日期/时间。
- 已完成任务重新启动时，如果原 DDL 已经过期或不存在，会要求设置新 DDL。
- 已过期任务继续处理时，仍然要求设置新 DDL。

## v1.3.5

- 修复「重新启动任务 / 继续处理」按钮点了没反应的问题。
- 前端点击逻辑改为读取真实按钮，并弹出确认 / 新 DDL 输入。
- 后端 `reactivate` 接口改用 `request.get_json(silent=True)`，避免空 JSON body 报错。

## v1.3.4

- 已完成任务按钮改为「重新启动任务」，点击前会确认。
- 已过期任务按钮为「继续处理」。
- 重新启动 / 继续处理时必须输入新的截止日期，可选截止时间。
- 后端 `reactivate` 接口支持写入新的 `due_date` / `due_time`。

## v1.3.3

- 优化归档任务的按钮文案。
- 已完成任务重新启动前增加确认，避免手误。
- 已过期任务继续处理不额外确认。

## v1.3.2

- 修复过期任务恢复后立刻再次归档的问题。
- 恢复时如果原截止时间已经过去，会清空旧 DDL 时间，避免马上再次过期。

## v1.3.1

- 修复顶部日期选择器在暗色模式下原生图标看不清的问题。
- 顶部日期改为可见的自定义 📅 按钮。

## v1.3.0

- 新增 DDL 自动过期归档。
- `tasks` 表新增：
  - `archived_at`
  - `archived_date`
  - `archive_reason`
- 完成任务时同时写入归档字段。
- 过期任务会从「今日需要注意的 DDL」移到「归档的 DDL」。
- 页面开着时，DDL 到期后会自动重新加载一次，让后端完成归档。

## v1.2.6

## v1.2.0

- 每个未完成 DDL 增加进度条。
- 进度条根据任务创建时间到截止时间的比例计算。
- 自动更新只刷新 DDL 倒计时和进度条，不会清空正在输入的留言。

## v1.1.1

- 修复页面刷新后「今天已完成」不显示的问题。

- 不涉及数据库结构变化，只需覆盖 `static/index.html`。

## v1.1.0

- 修复新完成记录的时间显示为 UTC 的问题。
- 新完成记录改为用电脑本地时间写入 `created_at`。
- DDL 新增 `due_time` 字段，支持具体到小时和分钟。
- DDL 列表显示完整截止时间。
- DDL 倒计时支持小时 / 分钟级别。
- 兼容旧数据库：旧任务没有时间时仍然按日期截止显示。

- 按日期查看完成记录。
- 亮色 / 暗色切换。
- 暗色模式启动闪烁修复。
- 每日提醒设置。
