# Morning Board（醒来先看）

当前版本：`v1.10.11`

Morning Board 是一个本地运行的个人晨间看板，用来把每天最需要看的事情放在一个简单页面里：今天完成了什么、明天先看什么、DDL、提醒、临时想法，以及单独拆出来的期末复习面板。

项目默认使用 Flask + SQLite，不需要登录，也不依赖外部数据库。数据会保存在本机项目目录里的 `.db` 文件中。

## 主要功能

### Morning Board 首页

首页用于日常任务和低成本复盘，适合每天打开看一眼。

包含：

- 今日完成事项
- 明日提醒 / 明天先看什么
- DDL 任务与截止时间
- DDL 可手动选择“生活相关 / 学习相关”，并用柔和淡金色 / 紫色边框区分；分类标签固定显示在标题下方
- DDL 倒计时始终显示具体剩余时间，精确到天 / 小时 / 分钟，并按分钟自动刷新
- 单行输入框支持 Enter 快速提交；多行留言保留 Enter 换行，使用 Ctrl/Cmd + Enter 保存

- 移动端首页顶部日期区会自动重新排布，避免日期框和“回到今天 / 切换亮暗 / 复制今日存档”按钮横向挤压或溢出
- 小感想分组
- 深色 / 浅色主题
- 可选 Telegram 晚间提醒
- Telegram 晚间提醒会附带当天已完成事项、提醒日期和明天 DDL
- 可选跨天转点报告，触发时间跟“每日切换时间”一致
- Telegram 晚间提醒和跨天转点报告都支持外部模板文件
- 跳转 Exam Board 的入口

### Exam Board

Exam Board 是专门给期末复习用的页面，不和日常 DDL 混在一起。

每门课可以记录：

- 课程编号和课程名
- 考试日期与时间
- 考试地点 / 模式
- 考试时长
- cheatsheet 是否允许
- cheatsheet / 考试规则备注
- 复习重点
- 当前复习进度
- 下一步行动
- 复习 checklist
- 考试要求截图 / 证据图片

截图功能适合保存 Canvas、邮件、课程大纲、考试说明等原始截图。这样之后不用只靠自己手写备注，可以直接把截图作为考据放在课程详情里。

## 安装与启动

先进入项目目录：

```bash
cd morning_board
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动：

```bash
python app.py
```

默认访问：

```text
http://127.0.0.1:5001/
```

如果是在局域网或 Tailscale 里访问，可以使用终端里显示的实际地址，例如：

```text
http://100.97.142.99:5001/
```

Exam Board 页面：

```text
http://127.0.0.1:5001/exams
```

## 使用方式

### 添加 DDL

在首页的 DDL 区域填写任务、类别、日期和时间，保存后会出现在任务列表中。

DDL 类别目前分为：

- `生活相关`：例如清理房间、收快递、退货、买东西等。
- `学习相关`：例如作业、考试、quiz、复习节点、整理资料等。

两类 DDL 会使用同一套卡片组件，但用贴合卡片边框的柔和彩色描边区分：生活相关偏柔和淡金色，学习相关偏紫色。

日期和时间输入框右侧有自定义的 📅 / ⏰ 按钮，深色模式下也能看清。

### 添加课程

进入 `/exams`，在“添加课程”区域填写：

- 课程编号，例如 `COMP5046`
- 课程名，例如 `Natural Language Processing`
- 考试时间

添加后课程会出现在课程卡片里。点“打开详情”可以继续补充考试规则、复习重点和 checklist。

### 保存考试要求截图

在课程详情里找到“考试要求截图 / 铁证考据”。

支持两种方式：

1. 点击“选择文件”，选择一张或多张图片后点“上传截图”。
2. 点击粘贴区域后，直接 `Ctrl+V` / `Cmd+V` 粘贴剪贴板里的截图。

上传后，截图会显示为缩略图。点击缩略图会在当前页面弹出图片预览弹窗，不再新开页面；不需要的截图可以删除。

支持格式：

```text
png, jpg, jpeg, webp, gif
```

默认单次请求大小上限约为 20 MB。

## Telegram 模板文件

Telegram 文案模板放在项目根目录，方便直接修改，不需要进代码里找。

```text
telegram_message_templates.txt
telegram_rollover_report_templates.txt
```

其中：

- `telegram_message_templates.txt`：晚间提醒模板。
- `telegram_rollover_report_templates.txt`：跨天转点报告模板。

模板标签只有在“单独一整行”时才会被识别，例如：

```text
[with_done]
...
[/with_done]
```

写在注释里的 `[with_done]`、`[without_done]`、`[ddl_section]`、`[rollover_with_done]` 等文本不会被当成模板标签。

跨天转点报告的触发时间会跟“每日切换时间”一致。如果每日切换时间设为 `05:00`，报告也会按 `05:00` 作为一天的分界发送。

## 数据文件说明

运行后会在项目目录生成：

```text
morning_board.db
exam_board.db
```

其中：

- `morning_board.db`：首页日常事项、DDL、小感想、设置等数据。
- `exam_board.db`：Exam Board 课程、复习 checklist、截图索引等数据。

考试截图图片会保存在：

```text
static/uploads/exam_evidence/
```

如果要备份完整数据，建议一起备份：

```text
morning_board.db
exam_board.db
static/uploads/exam_evidence/
```

## 升级注意事项

覆盖升级时，一般只覆盖代码文件：

```text
app.py
static/index.html
static/exams.html
telegram_message_templates.txt
telegram_rollover_report_templates.txt
README.md
CHANGELOG.md
```

不要删除这些本地数据：

```text
morning_board.db
exam_board.db
static/uploads/exam_evidence/
```

如果你删除数据库，之前保存的任务、课程和截图记录会丢失；如果只删除截图文件夹，数据库里仍可能保留截图记录，但图片会打不开。

## 常见问题

### 为什么看不到 Exam Board？

确认服务已经重启，并访问：

```text
http://127.0.0.1:5001/exams
```

### 为什么考试数据库不在项目目录？

v1.8.4 已经把 `exam_board.db` 固定到 `app.py` 所在项目目录，避免从不同目录启动时数据库跑偏。

### 为什么截图上传后看不到？

先确认：

- 图片格式是否为 png / jpg / jpeg / webp / gif。
- 单次上传文件是否太大。
- 项目目录下是否有 `static/uploads/exam_evidence/` 文件夹。
- 浏览器是否缓存了旧页面，可以强制刷新一次。

### 能不能直接粘贴截图？

可以。进入课程详情后，点击粘贴区域，再按 `Ctrl+V` / `Cmd+V`。如果剪贴板里是图片，会自动上传。
